# Support

Use GitHub issues for non-security support requests, bug reports, and feature proposals.

When reporting a runtime problem, include:

- macOS version
- `container system version`
- `container k8s version`
- command that failed
- relevant non-secret logs
- whether the cluster was created from a release, snapshot, or source build

Do not include kubeconfig credentials, tokens, private registry credentials, or private image names in public issues.
